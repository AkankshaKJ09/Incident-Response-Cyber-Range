# 🚀 Cyber Range Activity 2

## Incident Response Automation Cyber Range

---

## 📌 Overview

This project implements a **Cyber Range as a Service (CRaaS)** simulation using Python.
It provides an **interactive environment** where users can perform cyber attacks and observe **real-time detection, automated response, and forensic logging** through a graphical interface.

---

## 🎯 Objective

To develop a GUI-based cyber range simulation system that:

* Allows **interactive attack execution**
* Detects suspicious activities in real-time
* Performs **automated incident response**
* Generates logs and reports for analysis

---

## ⚙️ Features

* 🔴 Brute Force Attack Simulation (login attempts)
* 🟠 Malware Execution Simulation
* 🟢 Data Exfiltration Simulation
* ⚡ Real-time Detection Engine
* 🛡️ Automated Response System
* 📊 GUI-based Monitoring Dashboard
* 🧾 Incident Report Generation
* 🗂️ Forensic Logging using JSON

---

## 🧠 System Workflow

```
User Action → Log Generation → Detection Engine → Response Engine → GUI Display
```

---

## 🧩 Modules

* **Attack Simulator** – Generates logs based on user actions
* **Detection Engine** – Identifies suspicious patterns
* **Response Engine** – Triggers automated actions
* **Forensic Module** – Stores logs for analysis
* **Report Generator** – Creates incident reports
* **GUI (Tkinter)** – Displays system activity

---

## 🛠️ Technologies Used

* Python
* Tkinter (GUI)
* JSON (logging)
* GitHub (version control)

---

## ▶️ How to Run

1. Clone the repository:

```
git clone https://github.com/AkankshaKJ09/Incident-Response-Cyber-Range.git
```

2. Navigate to project folder:

```
cd Incident-Response-Cyber-Range
```

3. Run the application:

```
python main.py
```
